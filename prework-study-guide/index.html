HTML

  • The head element contains information about the webpage.
  • The Meta element contains the information for the browser.
  • The body element represents the visible content shown to the user.
  • H1 is only used once in a page, the rest are used in order to benefit screen readers.
  • Image information sits inside the image element brackets.
  • Viewport helps the webpage to adapt and fit the screen of the device.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Colours can be added as hex codes or standard references such as blue or yellow.
  • by linking the CSS file in the head section of the HTML we can effectively keep the files separate.

Git

  • Before pushing your changes to the main branch always pull the main branch in to check for changes - do this with git pull origin main.
  • Push your changes to the main branch with git push origin "branch name".
  • Always create a separate branch for any new feature or amend.
  • When working with a team, a pull request allows other users to check and review your code before you merge.

JavaScript

  • Javascript uses code flow to execute code, as a result code is actioned in order, so code needs to be added in logically.
  • Variables are containers for information in Javascript.
  • For loops can be used (as an example) to run through a numerical list in order, without needing to type code for each number.
  • Functions allow you to action a chunk of code at once with a function call, saving you from typing out all the code each time.
